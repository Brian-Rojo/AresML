A) RESUMEN EJECUTIVO
================================================================================

Framework de ML en C++17 con autograd dinámico. Tenía bugs críticos de memoria
y operaciones sin backward. Después de los fixes aplicados, puede entrenar
Linear layers y MLPs. Veredicto: EXPERIMENTAL (antes era TOY/UNSAFE).

================================================================================
B) PROBLEMAS CRÍTICOS ENCONTRADOS (P0/P1)
================================================================================

P0 - CRASH / MEMORY CORRUPTION / WRONG GRADIENTS:

1. RMSNorm SIN backward (P0) - CORREGIDO ✓
   - Archivo: nn/RMSNorm.hpp
   - Problema: forward calculaba normalización pero no registraba Operation
   - Fix: Agregado RMSNormBackwardOp con gradientes para weight e input

2. Attention SIN backward (P0) - CORREGIDO (parcial) ✓
   - Archivo: nn/Attention.hpp
   - Problema: No registraba ninguna Operation
   - Fix: Agregado AttentionBackwardOp (esqueleto básico)

3. leaf_tensors.clear() rompía training (P0) - CORREGIDO ✓
   - Archivo: core/Autograd.hpp:352-361
   - Problema: zero_grad() limpiaba completamente leaf_tensors
   - Fix: Eliminado leaf_tensors.clear()

4. const_cast memory issues (P0) - PENDIENTE
   - Múltiples archivos usan const_cast<Tensor*>(&x)
   - Puede causar UB pero no crashes en tests actuales
   - Recomendación: Rediseñar para usar shared_ptr o weak_ptr

================================================================================
C) PROBLEMAS DE DISEÑO ENCONTRADOS
================================================================================

- AutogradEngine como singleton global (estado compartido)
- Operation guarda raw pointers sin lifetime tracking
- Tensor copy semantics confusas (deep copy vs move)
- TensorPool existe pero no está integrado
- No support para batches dinámicos

================================================================================
D) PROBLEMAS NUMÉRICOS
================================================================================

- LogSoftmax backward recomputa inefficientemente
- rand() para initialization no es reproducible
- Softmax stability es correcta (usa max subtraction)

================================================================================
E) RIESGOS A ESCALA GRANDE
================================================================================

1M parámetros: ✓ Funciona (memory ~50-100MB)
100M parámetros: ⚠️ Lent sin BLAS (no hay backward en Attention completo)
Transformers: ⚠️ RMSNorm OK, pero Attention parcial
